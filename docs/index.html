night-worcoon-3

A small, hackable middleware proxy (HTTP + WebSocket) with a pluggable pipeline, a built-in mock and recorder, and a terminal UI for picking profiles at runtime.

TL;DR for the impatient It's a Node.js reverse proxy where each request flows through a chain of plugins. Plugins can short-circuit with a fake response (mock), replay an old recording, throttle traffic, or save real responses to disk. You pick which plugins are active per profile (one JSON file in configs/).

Quick start

npm install
npm start                                       # launch the TUI
npm run start:headless -- --config httpbin.json # no TUI

Then point your client at http://localhost:<port> (the port comes from the chosen profile).

What can I do with it?

Mock APIs

Return canned responses for chosen routes without touching the upstream. Great for frontend dev when the backend is broken.

Record & replay

Capture real upstream traffic, then replay it offline. Filesystem or SQLite backend.

CRUD bucket

Pretend to be a REST API. Drop a config, get POST/GET/PATCH/PUT/DELETE on any path with persistence.

Chaos test

Inject latency and random 5xx failures to stress-test client retry logic.

CORS layer

Slap CORS headers on any upstream and short-circuit preflights — even if the upstream doesn't.

WebSocket passthrough

ws:// upgrades on the listening port get forwarded to the target. (Plugin hooks don't see WS traffic yet.)

Project layout

configs/        profile JSONs (one per proxy)
plugins/        auto-loaded plugins (cors, latency, bucket, mock, recorder, your own)
recordings/     default storage dir (gitignored)
src/
  index.js      entry — picks TUI or --headless
  tui.js        blessed UI (Configs / Logs / Mocks / Plugins / Help tabs)
  proxy.js      HTTP + WebSocket proxy + plugin pipeline runner
  plugins.js    plugin loader (auto-discovers files in plugins/)
  storage.js    filesystem + sqlite recording backends
  match.js      URL/rule matching helpers (exact, substring, ":id" regex)
  logger.js     EventEmitter-based logger
docs/           you are here
