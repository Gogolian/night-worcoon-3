The TUI

TL;DR Run npm start and you get a terminal UI built on blessed with five tabs: Configs, Logs, Mocks, Plugins, Help. The proxy itself runs in the same Node process; the TUI just controls it.

Launching

npm start                                       # TUI (this page)
npm run start:headless -- --config httpbin.json # no TUI, just the proxy

Tabs

Configs

Browse every .json in configs/, see its details, and start / stop a proxy from the chosen profile.

Logs

Live tail of the shared logger (info / warn / error / debug) plus per-request trace events.

Mocks

Browse stored recordings from the active profile's storage, inspect headers/bodies, edit rules.

Plugins

Toggle plugins on/off for the active profile and reorder them.

Help

Quick keybinding cheat-sheet, in-app.

Global keybindings

KeyAction
← / →Switch tabs (when focus is on the tab bar)
↓ Drop focus into the active tab's content
↑ From the top of a tab, move focus back to the tab bar
Enter Activate the focused thing (start a proxy, edit a field, …)
s Stop the active proxy
Ctrl-R Reload the configs list from disk
c (Logs tab) Clear the log buffer
q / Ctrl-CQuit (stops the active proxy first)

The bottom status bar always shows the keys that apply to the currently focused panel. Watch it — that's the source of truth.

How the TUI talks to the proxy

src/index.js creates the TUI and passes in two callbacks:

createTui({
  configsDir, pluginsDir, logger,
  onStart: async (config) => startProxy({ config, ... }),  // returns handle
  onStop:  async (handle) => handle.stop(),
});

That's it. The TUI never talks to HTTP directly — it only spins up / tears down a startProxy() handle and listens to the same logger for live updates.

Want to run without a terminal? Use npm run start:headless -- --config <file.json>. Logs go to stdout, traces (one line per request) to stdout, SIGINT / SIGTERM shut down cleanly.
